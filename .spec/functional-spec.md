# 千穗梨衣_lily 歌单网站 — 功能规格书

## 1. 项目概述

为 Bilibili 主播 **千穗梨衣_lily**（UID: 279148275）打造的个人歌单展示网站。核心功能是一个可搜索、可筛选的歌曲目录，供粉丝在直播时点歌使用。

**核心原则：**
- 全站中文（简体），不使用英文术语
- 主播可自行管理歌单，无需开发者介入
- 视觉风格参考 diehikari.top，以浅紫色为主色调
- 架构支持未来扩展（粉丝账号、积分兑换等）

---

## 2. 功能需求

### 2.1 歌单展示（首页）

**页面结构：**
- 顶部：主播头像 + 名称 + 简介/标语
- 搜索栏：支持歌名、歌手名搜索，**支持拼音搜索**（如输入 "faruxue" 可匹配 "发如雪"）
- 筛选栏：按语言、分类标签筛选（标签从歌曲数据中动态生成）
- 歌曲列表：卡片式展示，无限滚动加载（每次约 30 首）

**每首歌展示：**
- 歌曲名称（点击复制到剪贴板）
- 歌手/原唱
- 专辑封面（可选，无封面时显示默认图）
- 语言标签
- 分类标签
- 点赞数（❤️）
- 添加日期

**搜索行为：**
- 实时搜索：用户输入时即时过滤结果
- 拼音匹配：预计算每首歌名和歌手名的拼音，存储于数据库
- 支持拼音首字母匹配（如 "frx" 匹配 "发如雪"）
- 同时匹配歌名和歌手名

**筛选行为：**
- 语言筛选：国语 / 英语 / 日语 / 韩语
- 分类标签筛选：从现有歌曲数据中动态提取，不硬编码
- 筛选与搜索可同时使用

**点赞行为：**
- 匿名点赞，无需登录
- 同一 IP 对同一首歌限制点赞次数（防刷）
- 点赞数实时显示

### 2.2 管理后台

**访问方式：**
- 独立的 `/admin` 路径
- 用户名 + 密码认证（存储于环境变量，密码哈希存储）
- 登录接口限流：同一 IP 连续失败 5 次后锁定 15 分钟

**歌曲管理 — 批量编辑模式：**

管理后台采用**表格式批量编辑**，而非逐条添加：

- 主播看到的是一个类似表格的界面，可同时编辑多行
- 新增歌曲：在表格底部添加新行，可一次添加多首
- 编辑歌曲：直接在表格中修改任意字段
- 删除歌曲：勾选行后批量删除，带确认提示
- **暂存机制**：所有修改先暂存在本地，点击"保存"后一次性提交到数据库
- **重复检测**：新增/编辑时，如果歌曲名称+歌手与已有记录重复，高亮警告（允许强制保存，因为可能是不同版本）

**数据导入/导出：**
- 导出：将当前歌单导出为 CSV 或 TSV 文件
- 导入：上传 CSV/TSV 文件批量导入歌曲，导入前预览并检测重复
- 导入/导出格式统一，方便在 Excel/WPS 中编辑

**版本管理：**
- 每次"保存"操作自动创建一个版本快照
- 保留最近 50 个版本
- 可查看历史版本列表（时间 + 变更摘要，如"新增 3 首，修改 2 首，删除 1 首"）
- 可回滚到任意历史版本（回滚本身也会创建新版本）

**分类标签管理：**
- 添加新标签时，直接在歌曲表单中输入即可（自动收集）
- 无需单独的标签管理页面

## 3. 数据模型

### 歌曲表 (songs)

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | INTEGER | 自动 | 主键，自增 |
| 歌曲名称 (song_name) | TEXT | ✅ | |
| 歌手 (artist) | TEXT | ✅ | 原唱/歌手名 |
| 语言 (language) | TEXT | ✅ | 国语/英语/日语/韩语 |
| 分类标签 (tags) | TEXT | ❌ | JSON 数组，如 ["流行","二次元"] |
| 专辑封面 (cover_url) | TEXT | 自动 | 通过 API 根据歌名+歌手自动获取，可手动覆盖 |
| 备注 (notes) | TEXT | ❌ | 特殊说明 |
| 排序权重 (sort_weight) | INTEGER | ❌ | 默认 0（保留字段，暂不使用） |
| 添加日期 (created_at) | TEXT | 自动 | ISO 日期字符串 |
| 点赞数 (likes) | INTEGER | 自动 | 默认 0 |
| 拼音_歌名 (pinyin_name) | TEXT | 自动 | 预计算的歌名全拼 |
| 拼音_歌手 (pinyin_artist) | TEXT | 自动 | 预计算的歌手名全拼 |
| 首字母_歌名 (initials_name) | TEXT | 自动 | 预计算的歌名拼音首字母 |
| 首字母_歌手 (initials_artist) | TEXT | 自动 | 预计算的歌手名拼音首字母 |

### 点赞记录表 (likes_log)

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER | 主键 |
| song_id | INTEGER | 外键 → songs.id |
| ip_hash | TEXT | IP 地址哈希（隐私保护） |
| created_at | TEXT | 点赞时间 |

### 版本快照表 (song_versions)

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER | 主键 |
| snapshot | TEXT | 完整歌单的 JSON 快照 |
| summary | TEXT | 变更摘要，如"新增 3 首，修改 2 首" |
| created_at | TEXT | 保存时间 |

## 4. 技术架构

### 前端
- **框架**: Next.js（App Router）
- **样式**: Tailwind CSS
- **部署**: Cloudflare Pages

**选择理由：**
- Next.js：生态成熟，组件丰富，SSR/SSG 灵活，参考站也使用 Next.js
- Tailwind CSS：快速迭代样式，后期主题切换方便
- Cloudflare Pages：免费额度充足，国内访问速度可接受

### 后端
- **运行时**: Cloudflare Workers（与 Pages 集成）
- **数据库**: Cloudflare D1（SQLite at edge）
- **拼音库**: pinyin-pro（在 Worker 端预计算拼音）

### API 端点

| 路径 | 方法 | 说明 |
|---|---|---|
| /api/songs | GET | 获取歌曲列表（支持分页、搜索、筛选） |
| /api/songs/batch | POST | 批量保存（新增+修改+删除，需认证） |
| /api/songs/:id/like | POST | 点赞（IP 限流） |
| /api/songs/export | GET | 导出歌单（CSV/TSV，需认证） |
| /api/songs/import | POST | 导入歌单（CSV/TSV，需认证） |
| /api/songs/cover | GET | 根据歌名+歌手获取专辑封面 |
| /api/tags | GET | 获取所有已用标签（从歌曲数据聚合） |
| /api/auth/login | POST | 管理员登录（限流） |
| /api/versions | GET | 获取版本历史列表（需认证） |
| /api/versions/:id | POST | 回滚到指定版本（需认证） |

## 5. 视觉设计方向

- **参考**: diehikari.top 的整体布局和风格
- **主色调**: 浅紫色系
- **风格**: 简洁、现代，卡片式布局
- **响应式**: 移动端优先（粉丝大多用手机看直播时点歌）
- **封面图**: 通过 API 自动获取专辑封面（参考 Firefly API 或其他音乐数据源），无封面时显示默认图

> ⚠️ **路线图项目**：后续与主播沟通确认具体视觉偏好后，再做风格定制。v1 先以参考站风格为基础。
